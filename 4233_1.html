Thanks for investing this issue! My guess is that in

jabref/src/main/java/org/jabref/gui/mergeentries/MergeEntries.java

Line 443 in 611ac55

entryPreview.setEntry(mergedEntry);

entryPreview is null. It is initialized in the constructor

jabref/src/main/java/org/jabref/gui/mergeentries/MergeEntries.java

Lines 182 to 187 in 611ac55

DefaultTaskExecutor.runInJavaFXThread(() -> {
entryPreview = new PreviewPanel(null, null, Globals.getKeyPrefs(), Globals.prefs.getPreviewPreferences(), new FXDialogService());
entryPreview.setEntry(mergedEntry);
JFXPanel container = CustomJFXPanel.wrap(new Scene(entryPreview));
mainPanel.add(container, CELL_CONSTRAINTS.xyw(1, 8, 6));
});

but this happens on a different thread/async, while the call

jabref/src/main/java/org/jabref/gui/mergeentries/MergeEntries.java

Line 206 in 611ac55

updateAll();

is immediate. If this analysis is correct indeed, then I would suggest to simply move the updateAll call to the end of runInJavaFXThread code.
